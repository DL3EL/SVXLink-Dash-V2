beim Raspberry Zero ist es sinnvoll, dieses so zu konfigurieren, dass er einen eigenen Hotspot aufspannt, wenn er keine Netzwerkverbindung findet
# Dazu wird ein eigener Hotspo konfiguriert
sudo nmcli connection add type wifi ifname wlan0 mode ap con-name "DL3ELHotspot" ssid "el-spot" autoconnect yes
# Passwort setzen (WPA2)
sudo nmcli connection modify "DL3ELHotspot" wifi-sec.key-mgmt wpa-psk wifi-sec.psk "elspot_dl3el"

# IP-Bereich und DHCP (Shared-Modus) festlegen
sudo nmcli connection modify "DL3ELHotspot" ipv4.method shared ipv4.addresses 192.168.50.1/24

# Priorität auf NIEDRIG setzen (Hausnetz ist 100, Hotspot ist 0)
sudo nmcli connection modify "DL3ELHotspot" connection.autoconnect-priority 0

# Eigenes WLAN mit hoher Prio eintragen. damit de Raspberry nach Neustart dorthin zurückkehrt. Den Namen findet man mit 'nmcli con show'
sudo nmcli connection modify "DEIN_HAUS_WLAN" connection.autoconnect-priority 100

Test:
Timer setzen: sudo shutdown -r +5 (In 5 Minuten startet der Pi).
Test-Aktivierung: 
sudo nmcli connection up "DL3ELHotspot"
Check, erreichbar?
Erfolg? Dann sudo shutdown -c zum Abbrechen des Neustarts.
Fehler? 5 Minuten warten, der Pi "rettet" sich selbst durch den Reboot.
================
